# HotPulse — AI 驱动的全球热点聚合平台

## 产品设计文档

> **定位：** 你的 AI 新闻副驾驶 — 不只是聚合，而是理解
> **口号：** News · Mindset · Wealth — 一个窗口看懂世界

---

## 一、产品概述

### 背景

热点新闻聚合市场竞争激烈（Google News、今日头条等），但大多数只是简单的信息搬运。HotPulse 的差异化在于：

1. **国内优先**：接入"每天60秒读懂世界"API + 微博热搜，国内内容优先展示
2. **AI 独立模块**：AI 分析作为独立视图，用户可切换查看原始新闻或 AI 增值处理结果
3. **数据可视化**：分类分布图、热度 Top10、股市折线图、AI 市场分析报告
4. **全球视野**：同时覆盖国内 + 国际 6+ 个平台，一键切换

### 核心价值

- **节省时间：** AI 摘要让你 3 秒看懂一条热点
- **国内优先：** 60s 每日精选 + 微博热搜，第一时间了解国内大事
- **全球视野：** 一键切换国际新闻，看遍 Reddit、Hacker News、YouTube、Twitter/X
- **数据洞察：** 分类分布、热度排行、AI 分析报告，不只是看新闻更是理解趋势
- **财富感知：** 股市实时指数 + 30/90 天折线图 + AI 市场分析
- **心态建设：** 每日励志金句，正能量陪伴

### 目标用户

- 国内互联网用户为主（默认国内视图）
- 关注科技、财经、时事的互联网从业者
- 投资者（股市模块 + AI 市场分析）
- 信息密集型工作者（需要高效获取信息）

## 三、功能详情

### 3.1 新闻模块

**页面路由：** `/`

**视图切换：**
- **全部新闻 / AI 分析** — 顶部 Tab 切换
- **国内 / 国际 / 全部** — 区域筛选（仅在"全部新闻"视图下显示）

**"全部新闻"视图（默认）：**
- 展示原始爬取的新闻（标题 + 内容摘要 + 平台 + 时间）
- 不显示 AI 摘要和情感标签
- 支持分类筛选（科技 / 财经 / 娱乐 / 体育 / 健康 / 其他）
- 默认显示国内新闻，可切换国际

**"AI 分析"视图：**
- 顶部统计图表：
  - 分类分布柱状图（各分类新闻数量对比）
  - 热度 Top 10 排行
- 卡片展示 AI 摘要 + 分类标签 + 情感标签 + 原始内容

**FeedCard 两种模式：**
- 原始模式：标题 + content_snippet + 平台 + 时间
- AI 模式：标题 + AI 摘要 + 分类 + 情感 + 内容 + 热度

### 3.2 AI 分析功能

调用 **智谱 AI（glm-4-flash）** 实现：

- [x] **一句话摘要：** 每条热点自动生成中文 + 英文各一句话摘要
- [x] **自动分类：** AI 判断热点所属分类
- [x] **情感分析：** 正面 / 负面 / 中性
- [x] **市场分析报告：** cron 预生成，涵盖各指数总结 + 趋势预测 + 风险提示
- [x] **批量处理：** 后端定时任务每小时抓取 + 批量 AI 处理

### 3.3 励志内容模块

- [x] 首页顶部展示每日金句（中英双语）
- [x] 预设 500+ 条名言库
- [x] 每次访问仅加载一次（全局 Context 管理）
- [x] 导出为 PNG 图片（白色背景，蓝色强调）
- [x] 分享到 Twitter / 微博 / 复制链接

### 3.4 股市模块

**页面路由：** `/stocks`

**视图切换：** 行情数据 | AI 分析

**"行情数据"视图：**
- 4 个全球主要指数：上证指数 / 纳斯达克 / 恒生指数 / 标普 500
- 每个指数卡片：价格（大字）+ 涨跌幅 + recharts 折线图
- 时间范围切换：7D / 30D / 90D
- CSV 导出

**"AI 分析"视图：**
- cron 预生成的 AI 市场分析报告
- 内容：各指数表现总结 + 市场趋势 + 风险提示 + 短期展望
- 中英双语

### 3.5 基础功能

- [x] **中英双语切换** — 全站 UI + 内容双语
- [x] **响应式设计** — 1/2/3/4 列自适应
- [x] **空状态处理** — 无数据时展示友好提示，不使用假数据
- [x] **SEO 优化** — Open Graph + Twitter Card meta
- [ ] **广告位** — Google AdSense（预留组件，流量达标后启用）

## 十三、风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 60s API 变更/不可用 | 国内新闻来源中断 | 多源冗余（微博 + 后续可接入知乎/头条） |
| 数据源 API 限流/变更 | 热点获取中断 | 多源冗余 + 空状态友好提示 |
| AI API 成本超预期 | 亏损 | glm-4-flash 低成本 + 市场分析预生成（非实时）|
| 微博/Twitter RSS 不稳定 | 数据缺失 | 优先保障稳定源（60s + Reddit/HN）|
| 版权合规风险 | 法律风险 | 仅聚合标题+摘要，不复制全文 |
| 用户增长缓慢 | 收入不足 | SEO + 社交分享 + AI 差异化内容 |
| Railway 免费额度耗尽 | 服务停机 | 控制 cron 频率，减少不必要请求 |
